# Pattern

Find and explain a specific conversion pattern from the knowledge base.

## Usage
```
/pattern [pattern-name]
```

## Examples
```
/pattern hero
/pattern social-proof
/pattern pricing-table
/pattern form-optimization
/pattern cta
/pattern faq
/pattern testimonials
```

## Instructions

When this command is invoked:

1. **Search for the pattern** in:
   - `library/sections/[pattern]/` - section patterns
   - `library/conversion/[pattern].md` - conversion patterns
   - `library/pages/*/` - page-level patterns

2. **Return comprehensive guidance**:

   ### Pattern Overview
   - What it is and when to use it
   - Key principles

   ### Structure
   - Recommended layout/wireframe (ASCII if available)
   - Required elements
   - Optional elements

   ### Best Practices
   - Do's and don'ts
   - Common mistakes to avoid
   - Mobile considerations

   ### Examples
   - Good vs bad examples
   - Industry-specific variations

   ### Data
   - Relevant benchmarks with sources
   - A/B test insights

3. **Include cross-references** to related patterns

## Available Patterns

### Sections
- `hero`, `cta`, `social-proof`, `testimonials`, `features`, `pricing`, `faq`, `stats`, `navigation`, `footer`, `carousel`

### Conversion
- `form-optimization`, `hero-patterns`, `cta-patterns`, `behavioral-economics`, `micro-conversions`

- `lift`, `meclabs`, `aida`, `pas`, `jtbd`, `value-proposition`
