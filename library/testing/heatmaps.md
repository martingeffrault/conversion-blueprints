# Heatmap Analysis Guide

> **Purpose**: Visualize user behavior to identify conversion opportunities
> **Impact**: Up to 250% conversion rate improvement (2% → 5%)
> **Principle**: See what users actually do, not what they say they do

## Types of Heatmaps

### Click Heatmaps (Click Maps)

**What they show**: Where users click (or try to click)

| Insight | Action |
|---------|--------|
| High clicks on non-clickable elements | Make them clickable |
| Low clicks on CTA | Redesign for visibility |
| Clicks on wrong elements | Fix misleading affordances |
| Dead zones | Rethink content placement |

> "Click heatmaps show where people actually click (or try to). Use them to see if your CTAs are working – or if users are getting sidetracked."
>
> *Source: [Heatmap.com](https://www.heatmap.com/blog/heatmap-analysis)*

### Scroll Heatmaps (Scroll Maps)

**What they show**: How far users scroll down the page

| Metric | Meaning |
|--------|---------|
| 100% view | Above fold (everyone sees) |
| 50% view | Average scroll depth |
| 25% view | Only engaged users reach |
| <10% view | Almost no one sees |

> "Scroll maps reveal where users stop scrolling. This information is vital for optimizing content placement and ensuring that your most important messages are seen."
>
> *Source: [SuperAGI](https://superagi.com/from-clicks-to-conversions-a-step-by-step-guide-to-using-ai-heatmaps-for-website-optimization/)*

### Move Heatmaps (Attention Maps)

**What they show**: Where users move their cursor (proxy for attention)

| Pattern | Interpretation |
|---------|----------------|
| High activity | Areas of interest/confusion |
| Reading patterns | F-pattern, Z-pattern |
| Hesitation zones | Confusion points |
| Ignored areas | Low-value content |

### Additional Heatmap Types

| Type | Purpose |
|------|---------|
| **Rage click maps** | Where users click repeatedly (frustration) |
| **Segment heatmaps** | Behavior by user segment |
| **Device heatmaps** | Mobile vs desktop differences |
| **Geo heatmaps** | Location-based patterns |

## Reading Heatmaps

### Color Interpretation

```
HOT (Red/Orange) ← Heavy interaction
                   High clicks/attention

WARM (Yellow)    ← Moderate interaction
                   Some engagement

COOL (Green/Blue)← Low interaction
                   Little engagement

COLD (No color)  ← No interaction
                   Completely ignored
```

### Common Patterns

**Click Map Patterns**

| Pattern | Meaning | Action |
|---------|---------|--------|
| Hot CTA | Working well | Don't change |
| Cold CTA | Not noticed | Redesign placement/design |
| Hot image | Visual interest | Add clickability |
| Hot text | Looks clickable | Make it a link |
| Scattered | No focus | Simplify page |

**Scroll Map Patterns**

| Pattern | Meaning | Action |
|---------|---------|--------|
| Steep drop at fold | Poor above-fold | Improve hero |
| Gradual decline | Normal | Optimize key points |
| Cliff at specific point | Content problem | Investigate that section |
| Deep scrolling | Engaged users | Content is working |

## Heatmap Analysis Process

### Step 1: Choose Pages to Analyze

| Priority | Pages |
|----------|-------|
| **High** | Homepage, landing pages, checkout |
| **Medium** | Product pages, category pages |
| **Lower** | Blog posts, about page |

### Step 2: Collect Sufficient Data

| Page Traffic | Minimum Sample |
|--------------|----------------|
| High (1K+/day) | 1-2 weeks |
| Medium (100-1K/day) | 2-4 weeks |
| Low (<100/day) | 4-8 weeks |

### Step 3: Segment Your Analysis

| Segment | Why |
|---------|-----|
| Device type | Mobile vs desktop behavior differs |
| Traffic source | Paid vs organic intent varies |
| New vs returning | Experience level matters |
| Converters vs non | What do buyers do differently? |

### Step 4: Identify Issues

```
FOR EACH HEATMAP:

1. What's getting attention?
   └── Is it what we WANT to get attention?

2. What's being ignored?
   └── Should it be ignored?

3. Where do users drop off?
   └── What's causing the drop?

4. Where do users click incorrectly?
   └── What's misleading them?

5. What patterns emerge?
   └── F-pattern, random, focused?
```

### Step 5: Prioritize Fixes

| Criteria | Weight |
|----------|--------|
| Impact on conversion | High |
| Ease of fix | Medium |
| Page traffic | High |
| Severity of issue | High |

## Page-Specific Analysis

### Homepage Heatmap Analysis

**Check for:**
- [ ] Is the value proposition getting attention?
- [ ] Is the primary CTA being clicked?
- [ ] Are users scrolling past the fold?
- [ ] Is navigation being used appropriately?
- [ ] Are secondary elements distracting from main goals?

### Landing Page Heatmap Analysis

**Check for:**
- [ ] Does attention flow to CTA?
- [ ] Is form getting engagement?
- [ ] Are trust signals being viewed?
- [ ] What percentage scroll to testimonials?
- [ ] Are there rage clicks anywhere?

### Product Page Heatmap Analysis

**Check for:**
- [ ] Are product images getting attention?
- [ ] Is add-to-cart visible and clicked?
- [ ] Are reviews being read?
- [ ] Is pricing clearly viewed?
- [ ] What information is being sought?

### Checkout Heatmap Analysis

**Check for:**
- [ ] Where do users hesitate?
- [ ] Are security badges noticed?
- [ ] Is form causing issues?
- [ ] Are payment options clear?
- [ ] Where do drop-offs occur?

## Combining Heatmaps with Other Tools

### Heatmaps + Session Recordings

| Heatmap Shows | Recordings Answer |
|---------------|-------------------|
| Many clicks here | Why are they clicking? |
| No scrolling past X | What stopped them? |
| Rage clicks | What frustrated them? |
| Unexpected pattern | What's the context? |

### Heatmaps + A/B Testing

```
WORKFLOW:

1. Heatmap analysis
   └── Identify problem areas

2. Form hypothesis
   └── "Moving CTA above fold will increase clicks"

3. A/B test
   └── Validate with statistical significance

4. Post-test heatmap
   └── Confirm behavior change
```

### Heatmaps + Surveys

| Heatmap Finding | Survey Question |
|-----------------|-----------------|
| Users ignoring testimonials | "What would help you trust us?" |
| Low scroll depth | "What info were you looking for?" |
| Hesitation at pricing | "What concerns do you have about pricing?" |

## Common Mistakes

### Analysis Mistakes

| Mistake | Consequence | Solution |
|---------|-------------|----------|
| Too small sample | Misleading patterns | Wait for sufficient data |
| Ignoring segments | Averaged-out insights | Segment by device, source |
| Only looking at clicks | Missing scroll/attention | Use all heatmap types |
| No hypothesis | Random optimization | Start with theory |
| Ignoring context | Misinterpretation | Combine with recordings |

### Implementation Mistakes

| Mistake | Consequence | Solution |
|---------|-------------|----------|
| Tracking without consent | Compliance issues | Implement consent |
| Heavy scripts | Slow page load | Async loading |
| Not masking fields | Privacy violation | Configure masking |
| Set and forget | Stale insights | Regular analysis |

## Heatmap Analysis Checklist

### Before Analysis

- [ ] Sufficient data collected (2+ weeks)
- [ ] Segments defined (device, source, etc.)
- [ ] Goals clear (what are we looking for?)
- [ ] Baseline metrics documented

### During Analysis

- [ ] Click map reviewed
- [ ] Scroll map reviewed
- [ ] Move/attention map reviewed
- [ ] Rage clicks checked
- [ ] Segments compared
- [ ] Patterns documented

### After Analysis

- [ ] Findings prioritized
- [ ] Hypotheses formed
- [ ] A/B tests planned
- [ ] Changes implemented
- [ ] Results measured

---

## Key Takeaways

1. **Click + Scroll + Move**: Use all heatmap types together
2. **Segment everything**: Mobile vs desktop, source, etc.
3. **Look for frustration**: Rage clicks reveal broken UX
4. **Focus on revenue**: Not just clicks, but conversions
5. **Combine with recordings**: See the "why" behind patterns
6. **Test your fixes**: Validate heatmap-driven changes with A/B tests
