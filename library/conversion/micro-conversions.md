# Micro-Conversions Framework

> **Purpose**: Track and optimize the small steps that lead to macro conversions.
> **Key insight**: Micro-conversions reveal funnel weaknesses before they hurt revenue.
> **Impact**: Reducing drop-off at micro-conversion points can double macro conversions.

---

## What Are Micro-Conversions?

> "Micro-conversions are the completion of small steps or secondary actions by a website visitor that indicate they are going to convert."
>
> *Source: [Hotjar](https://www.hotjar.com/conversion-rate-optimization/glossary/micro-conversion/)*

### Macro vs. Micro

| Type | Definition | Examples | Value |
|------|------------|----------|-------|
| **Macro** | Primary business goal | Purchase, signup, lead form | Direct revenue |
| **Micro** | Steps toward macro goal | Add to cart, page view, click | Leading indicator |

### Why Track Micro-Conversions?

> "If 60% of prospective customers are dropping off at a particular page in the purchase funnel, reducing that to 30% can drive a significant increase in conversions and revenue."
>
> *Source: [AgencyAnalytics](https://agencyanalytics.com/blog/micro-conversions)*

| Benefit | How It Helps |
|---------|--------------|
| Identify friction | See exactly where users drop off |
| Faster testing | More data points, quicker significance |
| Early warning | Spot problems before revenue drops |
| Optimization focus | Know which steps to improve first |

## Types of Micro-Conversions

### 1. Process Milestones

Direct steps toward the macro conversion.

> "Process milestones are the direct and trackable actions users take on their path toward a macro-conversion."
>
> *Source: [VWO](https://vwo.com/blog/what-are-micro-conversions-and-why-you-should-monitor-them/)*

**E-commerce:**
- Product page view
- Add to cart
- Begin checkout
- Enter shipping info
- Enter payment info
- Complete purchase ✓ (macro)

**SaaS:**
- View pricing page
- Start free trial
- Complete onboarding
- Use key feature
- Upgrade to paid ✓ (macro)

**Lead Generation:**
- Blog engagement
- Resource download
- Newsletter signup
- Contact form start
- Form submission ✓ (macro)

### 2. Secondary Actions

Engagement signals that indicate interest.

> "Secondary actions are indirect engagement signals that might not necessarily lead to a macro-conversion, but clearly indicate customer engagement and interest."
>
> *Source: [VWO](https://vwo.com/blog/what-are-micro-conversions-and-why-you-should-monitor-them/)*

| Action | What It Indicates |
|--------|-------------------|
| Newsletter signup | Interest, early funnel |
| Social follow | Brand affinity |
| Wishlist add | Purchase intent |
| Product comparison | High intent research |
| Live chat initiation | Help needed, could convert |
| Review reading | Purchase consideration |
| Video completion | Deep engagement |
| Multiple page views | Active exploration |

## Tracking Implementation

### Google Analytics 4 Events

```javascript
// E-commerce micro-conversions
gtag('event', 'view_item', {
  currency: 'USD',
  value: 29.99,
  items: [{ item_id: 'SKU123', item_name: 'Product' }]
});

gtag('event', 'add_to_cart', {
  currency: 'USD',
  value: 29.99,
  items: [{ item_id: 'SKU123', item_name: 'Product', quantity: 1 }]
});

gtag('event', 'begin_checkout', {
  currency: 'USD',
  value: 29.99,
  items: [{ item_id: 'SKU123' }]
});

// Custom micro-conversions
gtag('event', 'newsletter_signup', {
  method: 'footer_form'
});

gtag('event', 'pricing_view', {
  page_location: '/pricing'
});

gtag('event', 'demo_request', {
  form_location: 'pricing_page'
});
```

### Recommended Event Names

| Micro-Conversion | GA4 Event Name |
|------------------|----------------|
| Product view | `view_item` |
| Add to cart | `add_to_cart` |
| Remove from cart | `remove_from_cart` |
| Begin checkout | `begin_checkout` |
| Add shipping info | `add_shipping_info` |
| Add payment info | `add_payment_info` |
| Purchase | `purchase` |
| Signup | `sign_up` |
| Login | `login` |
| Search | `search` |
| Share | `share` |
| Custom | Use `snake_case` naming |

## Optimization Strategies

### 1. Identify Drop-Off Points

```
┌─────────────────────────────────────────────────────────────────┐
│  FUNNEL ANALYSIS                                                │
│                                                                 │
│  Step 1 → Step 2: 85% continue    ✓ Healthy                    │
│  Step 2 → Step 3: 70% continue    ✓ Acceptable                 │
│  Step 3 → Step 4: 35% continue    ⚠️ PROBLEM                   │
│  Step 4 → Step 5: 80% continue    ✓ Healthy                    │
│                                                                 │
│  Focus optimization on Step 3 → Step 4                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. A/B Test Micro-Conversions

> "Running A/B tests is key to optimizing micro-conversion stages. Try out different CTAs, messaging, and UI elements to see what boosts engagement."
>
> *Source: [Statsig](https://www.statsig.com/perspectives/micro-conversions-hidden-stages)*

| Test Element | Impact Area |
|--------------|-------------|
| CTA copy | Click rate |
| Button color/size | Click rate |
| Form length | Completion rate |
| Social proof placement | Trust, CVR |
| Progress indicators | Completion rate |
| Exit-intent offers | Save rate |

### 3. Reduce Friction

**E-commerce:**
- Guest checkout option
- Autofill addresses
- Multiple payment methods
- Save cart for later
- Clear shipping costs upfront

**SaaS:**
- Social login (Google, GitHub)
- Skip setup wizard option
- In-app onboarding
- Progress indicators
- Quick wins in first 5 minutes

**Lead Gen:**
- Fewer form fields
- Multi-step forms
- Conditional logic
- Prefill known data
- Clear value proposition

### 4. Add Motivation

| Tactic | Where to Use |
|--------|--------------|
| Urgency | Cart, checkout |
| Scarcity | Product page |
| Social proof | Throughout |
| Guarantees | Checkout, forms |
| Progress bars | Multi-step flows |
| Gamification | Onboarding |

## Common Issues & Fixes

### E-commerce

| Issue | Symptom | Fix |
|-------|---------|-----|
| Low ATC rate | High PDP views, low carts | Better CTAs, social proof, urgency |
| Cart abandonment | High cart, low checkout | Exit-intent offers, email recovery |
| Checkout drop-off | High checkout start, low complete | Guest checkout, fewer steps |
| Payment failure | High payment errors | Add payment methods, clear errors |

### SaaS

| Issue | Symptom | Fix |
|-------|---------|-----|
| Low trial starts | High pricing views, low signups | Reduce friction, social proof |
| Poor activation | Sign up but don't use | Better onboarding, quick wins |
| Trial abandonment | Active then inactive | In-app prompts, email sequences |
| Low conversion | Active but don't pay | Value demonstration, trial limits |

### Lead Gen

| Issue | Symptom | Fix |
|-------|---------|-----|
| Low form starts | High page views, low focus | Better CTA, form above fold |
| Form abandonment | Start but don't complete | Fewer fields, multi-step |
| Low-quality leads | High volume, poor quality | Better qualification questions |
| Email unsubscribes | High signup, high unsub | Better expectations, segmentation |

## Prioritization Framework

### ICE Score for Micro-Conversions

| Factor | Question | Score |
|--------|----------|-------|
| **I**mpact | How much will fixing this improve CVR? | 1-10 |
| **C**onfidence | How sure are we this will work? | 1-10 |
| **E**ase | How easy is this to implement? | 1-10 |

**ICE Score = (Impact + Confidence + Ease) / 3**

### Example Prioritization

| Micro-Conversion Fix | Impact | Confidence | Ease | ICE |
|---------------------|--------|------------|------|-----|
| Simplify checkout | 9 | 8 | 6 | 7.7 |
| Add exit-intent popup | 6 | 7 | 9 | 7.3 |
| Improve PDP images | 7 | 6 | 8 | 7.0 |
| Add progress bar | 5 | 8 | 9 | 7.3 |
| A/B test CTA color | 3 | 4 | 10 | 5.7 |
