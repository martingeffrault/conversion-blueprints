# Blog Article Page Blueprint

> **Purpose**: Educate, inform, build authority, drive conversions
> **Audience**: People searching for information, existing audience
> **Goal**: Rank in search, establish expertise, nurture leads

---

## Core Principle

Great blog articles answer questions better than anyone else. They're comprehensive, well-structured, and optimized for both readers and search engines. Every article should serve a purpose in your content strategy.

---

## Article Types

### Informational / Educational

**Purpose**: Answer questions, explain concepts
**Examples**:
- "What is [concept]?"
- "How does [thing] work?"
- "[Concept] explained"

### How-To / Tutorial

**Purpose**: Teach a process step-by-step
**Examples**:
- "How to [achieve goal]"
- "Complete guide to [topic]"
- "[Topic] tutorial for beginners"

### Listicle / Roundup

**Purpose**: Curate options, rank items
**Examples**:
- "[N] best [products] in [year]"
- "Top [N] tips for [goal]"
- "[N] ways to [achieve outcome]"

### Comparison / VS

**Purpose**: Help with decisions
**Examples**:
- "[Product A] vs [Product B]"
- "[Tool] alternatives"
- "Which [product] is right for you?"

### Case Study / Story

**Purpose**: Showcase results, build trust
**Examples**:
- "How [client] achieved [result]"
- "[Company] case study"
- "Behind the scenes: [project]"

### News / Updates

**Purpose**: Timeliness, thought leadership
**Examples**:
- "[Industry] trends for [year]"
- "What [update] means for [audience]"
- "[Event] recap"

### Introduction

**Structure:**
1. Hook (problem, question, or statistic)
2. Context (why this matters)
3. Promise (what reader will learn)
4. Transition to first section

**Length**: 100-200 words

### Body Sections

**Each major section (H2) should:**
- Cover one main idea
- Be scannable with subheadings
- Include visuals where helpful
- Link to relevant resources
- Be 300-500 words (guideline, not rule)

### Conclusion

**Structure:**
1. Recap key points
2. Reinforce main takeaway
3. Call-to-action (next step)

### Featured Image

- High-quality, relevant to topic
- Custom graphics > Stock photos
- Include alt text for SEO

| Type | Use Case |
|------|----------|

### Other Visual Elements

- **Pull quotes**: Highlight key insights
- **Callout boxes**: Tips, warnings, notes
- **Code blocks**: Technical content
- **Tables**: Comparisons, data
- **Embedded videos**: Tutorials, explanations

## SEO Considerations

### On-Page SEO

| Element | Best Practice |
|---------|---------------|
| Title tag | Include keyword, 50-60 chars |
| Meta description | Compelling summary, 150-160 chars |
| URL slug | Short, keyword-rich |
| H1 | One per page, includes keyword |
| H2-H3 | Include related keywords naturally |
| Alt text | Describe images, include keywords |
| Internal links | Link to related content |
| External links | Link to authoritative sources |

### Schema Markup

Use `Article` or `BlogPosting` schema:
- headline
- author
- datePublished
- dateModified
- image
- publisher

### Content Quality Signals

- Comprehensive coverage (answer all related questions)
- Original insights (not just rehashed content)
- Fresh data and examples
- Expert quotes or references
- Regular updates (for evergreen content)

- TOC is collapsible

## Engagement Elements

### Share Buttons

- Position: Top (for social-heavy content) or bottom
- Platforms: Focus on 3-4 relevant ones
- Include copy link option
- Show share counts (if impressive)

### Comments

**Consider:**
- Native vs. third-party (Disqus, etc.)
- Moderation requirements
- Spam handling
- Alternative: "Discuss on Twitter" link

### Reading Progress

- Progress bar at top of page
- Percentage indicator
- Helps long-form engagement
