# Product Launch / Waitlist Landing Page

> **Purpose**: Build anticipation and capture early interest for upcoming products
> **Conversion Goal**: Email signup (waitlist join)
> **SEO Priority**: Low (time-sensitive, traffic from owned channels)
> **Framework**: Curiosity + Exclusivity

---

## When to Use

| Launch Type | Page Focus |
|-------------|------------|
| **Pre-launch (months out)** | Teaser, brand building |
| **Coming soon (weeks out)** | Features revealed, urgency |
| **Waitlist (limited access)** | Exclusivity, early access benefits |
| **Beta launch** | Feedback-focused, community |

---

## SEO Considerations

**Should you SEO-optimize launch pages?**

Generally **no**:
- Temporary pages (expire at launch)
- Traffic from PR, social, ads
- Limited content (by design)
- Redirect to product page post-launch

**Exception**: Brand name searches â€” optimize for "[brand] launch" or "[product] waitlist".

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Too much information | Keep it minimal, build curiosity |
| No visual of product | Show a teaser/mockup |
| Weak value proposition | Be specific about the benefit |
| No social proof | Add waitlist count or early reactions |
| Missing privacy reassurance | Add "no spam" note |
| No post-signup engagement | Confirmation page + email |
