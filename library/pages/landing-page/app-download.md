# Mobile App Download Landing Page

> **Purpose**: Drive app store downloads for iOS/Android applications
> **Conversion Goal**: App install (click to app store)
> **SEO Priority**: Medium-High (brand + "app" keywords)
> **Framework**: Benefits-focused with social proof

---

## When to Use

| Scenario | Page Type |
|----------|-----------|
| **New app launch** | Feature-heavy, explainer |
| **Established app** | Social proof-heavy |
| **App store search** | SEO-optimized |
| **Paid campaigns** | Minimal, fast-loading |

---

## SEO Considerations

**Should you SEO-optimize app landing pages?**

| Keyword Type | SEO Value | Example |
|--------------|-----------|---------|
| Brand + app | ✅ High | "Spotify app download" |
| Category + app | ✅ High | "meditation app" |
| Competitor alternative | ⚠️ Medium | "[competitor] alternative app" |
| Generic app store | ❌ Low | "download app" (too broad) |

**Best practice**: Create dedicated landing pages for each target keyword (category pages, feature pages, vs competitor pages).

## Device Detection

### Smart Download Buttons

```html
<!-- Detect OS and show appropriate button -->
iOS user → Show App Store button prominently
Android user → Show Google Play button prominently
Desktop user → Show QR code + both store links
```

### QR Code Best Practices

- Link to smart link (detects device)
- Minimum size: 100x100px
- Include "Scan to download" label
- Test on multiple devices

## App Store Badge Guidelines

### Official Badges

| Store | Badge |
|-------|-------|
| Apple App Store | "Download on the App Store" |
| Google Play | "Get it on Google Play" |
| Amazon Appstore | "Available at Amazon" |

**Source**: [Apple Marketing Resources](https://developer.apple.com/app-store/marketing/guidelines/), [Google Play Badge Generator](https://play.google.com/intl/en_us/badges/)

### Badge Placement

- Always use official badges
- Maintain minimum clear space
- Don't alter colors or proportions
- Link directly to app store listing

## Mobile Optimization

Most app LP traffic is mobile. Priorities:

| Priority | Action |
|----------|--------|
| 1 | Fast load time (< 2s) |
| 2 | Prominent store badge |
| 3 | Sticky download CTA |
| 4 | Touch-friendly UI |
| 5 | Minimal form fields |

---

## Conversion Metrics

| Metric | Benchmark |
|--------|-----------|
| Click to app store | 20-40% |
| App store → Install | 30-50% (depends on ASO) |
| Overall LP → Install | 10-20% |

> **Note**: Track with deep linking (Branch, AppsFlyer, Adjust)

## Technical Considerations

| Requirement | Notes |
|-------------|-------|
| Deep linking | Smart links that detect device |
| Attribution | Track campaign → install |
| Preview cards | OG tags for social sharing |
| App Store SEO | Align LP keywords with ASO |
| Schema markup | SoftwareApplication schema |

### Schema Example

```json
{
  "@type": "SoftwareApplication",
  "name": "App Name",
  "operatingSystem": "iOS, Android",
  "applicationCategory": "LifestyleApplication",
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.8",
    "reviewCount": "10000"
  },
  "offers": {
    "@type": "Offer",
    "price": "0",
    "priceCurrency": "USD"
  }
}
```

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Generic screenshots | Show actual app UI |
| No visible store badges | Make badges prominent |
| Desktop-focused design | Mobile-first |
| Too much text | Visual-first approach |
| No rating shown | Display rating + review count |
| Slow load time | Optimize images, minimize scripts |
