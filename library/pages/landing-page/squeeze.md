# Squeeze Page (Email Capture)

> **Purpose**: Capture email addresses with minimal friction
> **Conversion Goal**: Email opt-in
> **SEO Priority**: Low-None (traffic from ads/referrals)
> **Framework**: Single-minded focus on one action

---

## When to Use

| Scenario | Page Type |
|----------|-----------|
| **Lead magnet giveaway** | Resource download |
| **Newsletter signup** | Content subscription |
| **Contest entry** | Sweepstakes/giveaway |
| **Course preview** | Free lesson access |
| **Tool access** | Free trial/demo |

---

## SEO Considerations

**Should you SEO-optimize squeeze pages?**

Generally **no**:
- Traffic comes from ads, email, social
- Minimal content (by design)
- Often noindex is recommended

**Exception**: Lead magnet squeeze pages can rank for "[topic] free guide/template/checklist" if there's search volume.

---

## Core Principles

### 1. Single Focus

- One offer, one action
- No navigation
- No distractions
- No external links

### 2. Minimal Friction

- Email only (or email + name max)
- Fast load time
- No complex forms

### 3. Clear Value Exchange

- Obvious benefit
- Immediate gratification
- Trust indicators

| Email + First name | -10-15% |
| Email + Full name | -15-20% |

## Visual Elements

### Lead Magnet Mockup

- Show the actual resource
- 3D mockup for ebooks/PDFs
- Screenshots for tools
- Video thumbnail for video content

### Image Guidelines

- High quality (no stock feel)
- Relevant to the offer
- Fast loading (optimize)
- Mobile-friendly display

---

## Mobile Optimization

Critical for squeeze pages (often 50%+ mobile traffic):

| Priority | Action |
|----------|--------|
| 1 | Single-column layout |
| 2 | Large, tappable button |
| 3 | Visible form without scrolling |
| 4 | Fast load time (< 2s) |
| 5 | Touch-friendly input |

## A/B Testing Ideas

| Element | Variations to Test |
|---------|-------------------|
| Headline | Benefit vs curiosity vs how-to |
| Image | With mockup vs without |
| Form fields | Email only vs email + name |
| Button text | Action words comparison |
| Layout | Split vs stacked |
| Trust elements | With vs without testimonial |

---

## Technical Checklist

| Item | Notes |
|------|-------|
| Form validation | Email format check |
| Thank you redirect | Immediate after submit |
| Email delivery | Trigger within seconds |
| Mobile responsive | Test on devices |
| Page speed | < 2 seconds load |
| No navigation | Remove header/footer links |
| Tracking | UTM, conversion pixels |

---

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Too many form fields | Email only (or + name max) |
| Weak headline | Be specific about the result |
| No mockup/visual | Show what they're getting |
| Distracting navigation | Remove all links |
| Slow page load | Optimize everything |
| No privacy note | Add trust text |
| Generic button text | Use action-oriented copy |
| No thank you page | Always confirm + deliver |
