# Pricing Section Best Practices

> **Purpose**: Present pricing options clearly to drive conversions
> **When to Use**: Pricing page, homepage (teaser), landing pages
> **Framework**: Comparison + Value Anchoring

---

## Pricing Display Strategies

### When to Show Pricing

| Scenario | Recommendation |
|----------|----------------|
| Standardized pricing | ✅ Always show |
| Tiered plans (SaaS) | ✅ Always show |
| Starting prices | ⚠️ "Starting at" approach |
| Custom/Enterprise | ❌ "Contact us" + value focus |
| Highly variable | ❌ "Get a quote" |

### Hide Pricing When

- Price requires context (custom solutions)
- Sales qualification needed
- Complex pricing models
- Negotiated deals expected

## Pricing Psychology

### Price Anchoring

Show value before price:

```
Value: $2,000+
Today: $499 (75% off)
```

### The Decoy Effect

Middle option should be the target:

| Tier | Price | Purpose |
|------|-------|---------|
| Basic | Low | Anchor, entry point |
| Pro | Medium | **Target** (most choose this) |
| Enterprise | High | Makes Pro seem reasonable |

### Payment Framing

| Frame | Example |
|-------|---------|
| Monthly | $79/month |
| Daily | "Less than $3/day" |
| Per user | $10/user/month |
| Annual (savings) | $790/year (save $158) |

## Key Elements

### Required Elements

| Element | Purpose |
|---------|---------|
| Plan name | Clear identification |
| Price | Obvious, prominent |
| Billing period | Monthly/annual clarity |
| Feature list | What's included |
| CTA button | Clear next step |

### Recommended Elements

| Element | When to Include |
|---------|-----------------|
| "Most Popular" badge | Highlight target plan |
| Savings callout | Annual vs monthly |
| Free trial mention | Reduce friction |
| Money-back guarantee | Risk reversal |
| Feature comparison | Differentiation |

### Optional Elements

| Element | Use Case |
|---------|----------|
| User count | Per-seat pricing |
| Toggle (monthly/annual) | Dual pricing |
| Currency selector | International |
| FAQ link | Complex pricing |

## Toggle (Monthly/Annual)

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│         [Monthly]  ◐───────────────────●  [Annual]             │
│                                         Save 20%                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**Best Practices**:
- Default to annual (higher value)
- Show savings amount ("Save 20%")
- Animate price change
- Clear which is selected

## Feature Lists

### Do's

- Lead with most valuable features
- Use clear, benefit-focused language
- Include checkmarks for included
- Show limits clearly (e.g., "5 users")
- Group related features

### Don'ts

- Don't list every feature
- Don't use jargon
- Don't make lists too long (7-10 max)
- Don't use ✗ excessively

### Feature Display

```
✓ Included feature
✗ Not included (or simply omit)
[5] Limited (show the limit)
∞ Unlimited
```

## CTA Buttons

### Button Text by Plan Type

| Plan | CTA |
|------|-----|
| Free | "Start Free", "Get Started" |
| Paid (trial) | "Start Free Trial" |
| Paid (no trial) | "Subscribe", "Buy Now" |
| Enterprise | "Contact Sales", "Talk to Us" |

### Visual Hierarchy

- **Primary plan**: Primary button style
- Other plans: Secondary/ghost style
- Enterprise: Outline or text link

## Trust Elements

Place near pricing:

| Element | Purpose |
|---------|---------|
| Money-back guarantee | Risk reversal |
| Security badges | Payment safety |
| Review/rating | Social proof |
| Customer logos | Validation |
| "No credit card required" | Trial friction |

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Too many tiers | Limit to 3-4 |
| Hidden pricing | Be transparent |
| No featured plan | Highlight recommended |
| Feature overload | Curate key features |
| Missing billing period | Show /month or /year |
| No annual option | Offer with savings |
| Unclear CTAs | Specific action text |
| No trust elements | Add guarantee, badges |
