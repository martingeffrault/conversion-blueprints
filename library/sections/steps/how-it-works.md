# Steps / How It Works Section

> **Purpose**: Explain process, reduce complexity, build confidence
> **Position**: After hero or features, before CTA
> **Goal**: Make the path to success clear and achievable

---

## Core Principle

People avoid what they don't understand. The "How it works" section demystifies your offering by breaking it into simple, sequential steps. Three is the magic number.

> "The Rule of Three works because people can remember three things easily. Any more and retention drops dramatically."
>
> *Source: UX writing best practices*

---

## Optimal Number of Steps

| Steps | When to Use | Notes |
|-------|-------------|-------|
| **3 steps** | Most situations | Ideal for cognitive load |
| **4 steps** | Complex processes | Maximum before confusion |
| **5+ steps** | Avoid if possible | Break into phases instead |

**Why 3 works:**
- Easy to remember
- Creates clear beginning, middle, end
- Feels achievable
- Fits naturally on screens

---

## Step Structure

Each step should have:

```
[Number or Icon]
Step Title (2-4 words)
Brief description (1-2 sentences)
```

### Step Anatomy

| Element | Required | Description |
|---------|----------|-------------|
| **Number/Icon** | Yes | Visual identifier (1, 2, 3 or icons) |
| **Title** | Yes | Action-oriented, starts with verb |
| **Description** | Yes | What happens, benefit delivered |
| **Visual** | Optional | Screenshot, illustration, icon |
| **Duration** | Optional | Time estimate if relevant |
