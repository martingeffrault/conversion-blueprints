# Features Section Best Practices

> **Purpose**: Communicate product/service capabilities and benefits
> **Position**: After hero, before social proof
> **Goal**: Help visitors understand what they get and why it matters

---

## Core Principle

Features sections bridge the gap between "what you do" and "why it matters." The best feature sections don't just list capabilities â€” they connect features to outcomes.

### The Feature-Benefit Connection

| Feature (What) | Benefit (Why It Matters) |
|---------------|--------------------------|
| 256-bit encryption | Your data is completely secure |
| AI-powered suggestions | Save 5 hours per week |
| One-click integration | Get started in minutes |
| Real-time sync | Your team is always aligned |

> "Don't sell features, sell the transformation. Features tell what, benefits tell why it matters to the customer."

---

## Writing Feature Copy

### Headline Formulas

| Formula | Example |
|---------|---------|
| Adjective + Noun | "Powerful Analytics" |
| Verb + Outcome | "Save Hours Every Week" |
| Benefit Statement | "Never Miss a Deadline" |
| For [Audience] | "Built for Remote Teams" |

### Description Best Practices

**Do**:
- Lead with the benefit
- Use active voice
- Be specific (numbers help)
- Keep it scannable

**Don't**:
- Use jargon
- Write paragraphs
- List specs without context
- Be vague

## Number of Features to Show

### Cognitive Load Considerations

| Count | Best For |
|-------|----------|
| 3 | Landing pages, focused messaging |
| 4 | Balanced overview |
| 6 | Comprehensive but scannable |
| 8+ | Tabbed or categorized display |

### The Rule of 3-6

Most feature sections work best with 3-6 items:
- **3 features**: High-level, memorable
- **4 features**: Balanced grid (2x2)
- **6 features**: Comprehensive (3x2 or 2x3)

For more features, use:
- Categories/tabs
- "See all features" link
- Separate features page

## Visual Elements

### Icons

- **Style**: Consistent set (outline, solid, or duotone)
- **Size**: Large enough to be recognized (32-48px)
- **Color**: Brand color or neutral with occasional accent
- **Purpose**: Quick visual identification, not decoration

### Images

- **Product screenshots**: Show actual UI
- **Illustrations**: Conceptual features
- **Photos**: Outcome/lifestyle imagery
- **Videos**: Complex features, demos

### Layout Considerations

- **Alignment**: Consistent across cards
- **Spacing**: Adequate breathing room
- **Hierarchy**: Headlines prominent, descriptions subdued
- **Grid**: Even columns, consistent card sizes (unless bento)

## Accessibility

### Feature Section Checklist

- [ ] Icons have aria-label or are decorative (aria-hidden)
- [ ] Color not sole indicator of meaning
- [ ] Text has sufficient contrast
- [ ] Links are descriptive ("Learn about analytics" not "Learn more")
- [ ] Images have alt text
- [ ] Keyboard navigation works for tabs

## Common Mistakes

### âŒ Feature Overload

Showing 12+ features creates cognitive overload. Focus on 6 key features or use categorization.

### âŒ Features Without Benefits

"256-bit encryption" means nothing without "Your data is secure from any threat."

### âŒ Generic Icons

Icons should aid understanding, not just fill space. Choose icons that relate to the feature.

### âŒ Wall of Text

Feature descriptions should be scannable. 1-2 sentences max per feature.

### âŒ Inconsistent Cards

Mixed card sizes, alignments, or content lengths look unprofessional.

### âŒ No Visual Hierarchy

Without clear hierarchy, visitors don't know what's most important.
