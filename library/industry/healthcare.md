## Trust Signals (Critical)

Healthcare requires the highest level of trust. Every element should reinforce credibility.

### Essential Trust Elements

| Element | Placement | Purpose |
|---------|-----------|---------|
| Physician credentials | Provider profiles | Expertise validation |
| Board certifications | Near name/photo | Authority signal |
| Hospital affiliations | Provider profiles | Institutional backing |
| HIPAA compliance badge | Footer, forms | Privacy assurance |
| Accreditations | About page, footer | Quality standards |
| Patient reviews | Provider profiles | Social proof |
| Years of experience | Provider profiles | Expertise |
| Insurance accepted | Homepage, booking | Accessibility |

### Patient Information Form

**Required fields** (HIPAA-conscious):
- Full legal name
- Date of birth
- Contact information
- Insurance information (optional at booking)
- Reason for visit
- New or existing patient

**Best practices**:
- Progress indicator
- Save and continue later
- Secure form badge visible
- Clear privacy statement
- Accessibility compliant inputs

## Accessibility Requirements

Healthcare sites have heightened accessibility obligations.

### WCAG Priorities

| Requirement | Implementation |
|-------------|----------------|
| Screen reader compatible | ARIA labels, semantic HTML |
| Keyboard navigation | Full tab navigation |
| Color contrast | 4.5:1 minimum (text), 3:1 (UI) |
| Font sizing | 16px minimum, resizable |
| Form labels | Explicit labels, not placeholders |
| Error messages | Clear, specific, helpful |
| Alternative text | Meaningful descriptions |
| Video captions | All video content |

### Specific Healthcare Accessibility

| Feature | Why Critical |
|---------|--------------|
| Large touch targets | Elderly users, motor impairments |
| High contrast mode | Vision impairments |
| Text-to-speech friendly | Vision impairments |
| Simple language | Cognitive accessibility |
| Print-friendly layouts | Offline reference |
