# Conversion & Behavior Frameworks

> **Best for**: Landing page optimization, UX design, behavior design, habit formation
> **Audience**: Marketers, product designers, growth teams
> **Focus**: Understanding and influencing user behavior

---

## Overview

These frameworks explain WHY people take action (or don't). Use them to diagnose conversion problems and design better experiences.

---

## LIFT Model

> **Origin**: WiderFunnel
> **Best for**: Landing page optimization, conversion audits
> **Core insight**: Conversion = Value Proposition × (Relevance + Clarity + Urgency) - (Anxiety + Distraction)

### The Framework

```
                      ┌─────────────────────┐
                      │ VALUE PROPOSITION   │
                      │   (The Offer)       │
                      └──────────┬──────────┘
                                 │
              ┌──────────────────┼──────────────────┐
              │                  │                  │
              ▼                  ▼                  ▼
       ┌──────────┐       ┌──────────┐       ┌──────────┐
       │RELEVANCE │       │ CLARITY  │       │ URGENCY  │
       │          │       │          │       │          │
       │ Is this  │       │ Do I     │       │ Why      │
       │ for me?  │       │understand│       │ now?     │
       │          │       │ it?      │       │          │
       └──────────┘       └──────────┘       └──────────┘
              │                  │                  │
              │         AMPLIFY THESE              │
              └──────────────────┬─────────────────┘
                                 │
              ┌──────────────────┴─────────────────┐
              │                                    │
              ▼                                    ▼
       ┌──────────┐                         ┌──────────┐
       │ ANXIETY  │                         │DISTRACTION│
       │          │                         │          │
       │ Can I    │                         │ What     │
       │ trust    │                         │ else is  │
       │ this?    │                         │ here?    │
       └──────────┘                         └──────────┘
              │                                    │
              │          REDUCE THESE             │
              └────────────────────────────────────┘
```

### Elements Deep Dive

#### Value Proposition (Foundation)

**Question**: "Why should I choose this over alternatives?"

| Strong Value Prop | Weak Value Prop |
|-------------------|-----------------|
| Specific outcome | Generic benefit |
| Quantified results | Vague claims |
| Unique differentiator | "Me too" positioning |

**Audit questions**:
- Is the core benefit crystal clear?
- Is there a compelling reason to choose you?
- Is it believable?

#### Relevance

**Question**: "Is this for me?"

| Signals of Relevance | Signals of Irrelevance |
|---------------------|------------------------|
| My industry/role mentioned | Generic messaging |
| Problems I recognize | Unfamiliar terminology |
| People like me shown | Stock photos |

**Audit questions**:
- Does the headline speak to my specific situation?
- Do I see myself in the content/imagery?
- Does the CTA match my intent?

#### Clarity

**Question**: "Do I understand what this is and does?"

| Clear | Unclear |
|-------|---------|
| Simple language | Jargon |
| Visual hierarchy | Wall of text |
| Logical flow | Scattered layout |

**Audit questions**:
- Can someone understand this in 5 seconds?
- Is there one clear next step?
- Is the design supporting or hindering comprehension?

#### Urgency

**Question**: "Why should I act now?"

| Real Urgency | Fake Urgency |
|--------------|--------------|
| Limited availability | "Order now!" |
| Deadline-driven | Countdown timer (always) |
| Scarcity proof | "Only 2 left" (fabricated) |

**Audit questions**:
- Is there a legitimate reason to act now?
- Have I communicated the cost of delay?
- Am I using urgency ethically?

#### Anxiety (Reduce)

**Question**: "Can I trust this?"

| Anxiety Reducers | Anxiety Creators |
|------------------|------------------|
| Testimonials | Suspicious claims |
| Trust badges | Hidden pricing |
| Clear policies | No contact info |
| Security icons | Long forms |

**Audit questions**:
- What might make someone hesitate?
- Have I addressed common objections?
- Is there social proof visible?

#### Distraction (Reduce)

**Question**: "What's competing for attention?"

| Low Distraction | High Distraction |
|-----------------|------------------|
| Single CTA focus | Multiple CTAs |
| Clean design | Cluttered layout |
| Supporting content | Tangential info |

**Audit questions**:
- Is there ONE clear action to take?
- Can anything be removed?
- Does everything support the goal?

### LIFT Audit Checklist

```
VALUE PROPOSITION
□ Core benefit is clear and compelling
□ Differentiation from alternatives is obvious
□ Claims are specific and believable

RELEVANCE
□ Headline speaks to target audience's situation
□ Imagery reflects target audience
□ Content addresses their specific problems

CLARITY
□ Can understand the offer in 5 seconds
□ Clear visual hierarchy
□ One obvious next step

URGENCY
□ Legitimate reason to act now
□ Cost of delay communicated
□ Not using manipulative tactics

ANXIETY
□ Trust signals present (reviews, badges, logos)
□ Objections addressed
□ Contact/support options visible

DISTRACTION
□ Single primary CTA
□ No unnecessary navigation
□ All content supports conversion goal
```

## Fogg Behavior Model

> **Origin**: BJ Fogg (Stanford)
> **Best for**: Behavior design, habit formation, reducing friction
> **Core insight**: Behavior = Motivation × Ability × Prompt (B=MAP)

### The Framework

```
         ▲
         │
   HIGH  │         ╔═══════════════════╗
         │         ║  ACTION LINE      ║
         │  ★ Easy ║  ──────────────── ║
 M       │  action ║  Prompts succeed  ║
 O       │  works  ║  above this line  ║
 T       │    here ║                   ║
 I       │         ╚═══════════════════╝
 V       │        ╱
 A       │       ╱ ← Action threshold
 T       │      ╱
 I       │     ╱
 O       │    ╱
 N       │   ╱
         │  ╱
         │ ╱ Prompts fail
         │╱  below this line
   LOW   │─────────────────────────►
              HARD           EASY
                  ABILITY
```

### The Three Elements

#### Motivation

What drives someone to want to take action:

| Motivator | Positive | Negative |
|-----------|----------|----------|
| **Sensation** | Pleasure | Pain |
| **Anticipation** | Hope | Fear |
| **Social** | Acceptance | Rejection |

**Increasing motivation**:
- Connect to existing desires
- Show others like them succeeding
- Make the outcome vivid

#### Ability

How easy is the action to complete:

| Simplicity Factor | Easy | Hard |
|-------------------|------|------|
| **Time** | Quick | Long |
| **Money** | Free/cheap | Expensive |
| **Physical effort** | Minimal | Exhausting |
| **Mental effort** | Intuitive | Complex |
| **Social deviance** | Normal | Unusual |
| **Routine** | Fits habits | Disrupts habits |

**Increasing ability**:
- Reduce steps
- Pre-fill information
- Provide defaults
- Break into smaller actions

#### Prompt

The trigger that initiates the behavior:

| Prompt Type | Description | Example |
|-------------|-------------|---------|
| **Facilitator** | High motivation, low ability | "We'll do it for you" |
| **Signal** | High motivation, high ability | Simple reminder |
| **Spark** | Low motivation, high ability | Motivating message |

**Effective prompts**:
- Noticed by the person
- Associated with the target behavior
- Delivered when motivation/ability are sufficient

### Applying Fogg to Web Design

#### Form Optimization

```
BEFORE: Long form (low ability)
┌────────────────────────────────────────┐
│ Name: _______________                  │
│ Email: ______________                  │
│ Phone: ______________                  │
│ Company: ____________                  │
│ Size: _______________                  │
│ Industry: ___________                  │
│ Budget: _____________                  │
│ Timeline: ___________                  │
│ How did you hear about us? __________ │
│ [Submit]                               │
└────────────────────────────────────────┘

AFTER: Minimal form (high ability)
┌────────────────────────────────────────┐
│ Email: ______________                  │
│ [Get Started]                          │
│                                        │
│ (We'll ask for more info later)        │
└────────────────────────────────────────┘
```

#### CTA Optimization

| Low Ability CTA | High Ability CTA |
|-----------------|------------------|
| "Sign up for free trial" | "Start free — no credit card" |
| "Contact sales" | "Get a call in 5 minutes" |
| "Download whitepaper" | "Get instant access" |

### Fogg Behavior Grid

Map behaviors by frequency and duration:

| | One-time | Short-term | Long-term |
|---|---|---|---|
| **New behavior** | Green Path (easiest) | Blue Path | Purple Path (hardest) |
| **Familiar behavior** | Requires prompt only | Habit building | Deep change |

**Start with Green Path behaviors** (one-time, new) to build momentum.

## Hook Model

> **Origin**: Nir Eyal
> **Best for**: Product design, habit formation, engagement
> **Core insight**: Habits form through repeated cycles of Trigger → Action → Variable Reward → Investment

### The Hook Cycle

```
         ┌──────────────────────────────────────────┐
         │                                          │
         ▼                                          │
    ┌─────────┐                              ┌─────────────┐
    │ TRIGGER │                              │ INVESTMENT  │
    │         │                              │             │
    │ External│                              │ User puts   │
    │ or      │                              │ something   │
    │ Internal│                              │ in          │
    └────┬────┘                              └──────┬──────┘
         │                                          ▲
         │                                          │
         ▼                                          │
    ┌─────────┐                              ┌─────────────┐
    │ ACTION  │──────────────────────────────►│   REWARD   │
    │         │                              │  (Variable) │
    │ Simplest│                              │             │
    │ behavior│                              │ Tribe/Hunt/ │
    │ in      │                              │ Self        │
    │ anticipation                           │             │
    │ of reward                              │             │
    └─────────┘                              └─────────────┘
```

### Phase 1: Trigger

What prompts the user to start the behavior:

| Trigger Type | Description | Example |
|--------------|-------------|---------|
| **External** | Outside prompt | Push notification, email |
| **Internal** | Inside prompt | Boredom, anxiety, FOMO |

**Goal**: Move from external triggers to internal triggers (habit formed).

**Common internal triggers**:
- Boredom → Social media
- Anxiety → Email
- Uncertainty → Google
- Loneliness → Messaging apps

### Phase 2: Action

The simplest behavior done in anticipation of reward:

**Apply Fogg's B=MAP**:
- Make the action as easy as possible
- Trigger when motivation is high
- Remove all friction

| App | Trigger | Action (Simple) |
|-----|---------|-----------------|
| Twitter | Boredom | Open app, scroll |
| Google | Question | Type, hit enter |
| Slack | FOMO | Click notification |

### Phase 3: Variable Reward

The unpredictable payoff that drives the behavior:

| Reward Type | Description | Example |
|-------------|-------------|---------|
| **Tribe** | Social rewards | Likes, comments, recognition |
| **Hunt** | Material rewards | Money, information, resources |
| **Self** | Personal achievement | Completion, mastery, competence |

**Key**: Variability creates engagement. Predictable rewards become boring.

### Phase 4: Investment

User puts something in that increases likelihood of return:

| Investment Type | Description | Effect |
|-----------------|-------------|--------|
| **Time** | Building profile, history | Stored value |
| **Data** | Preferences, content | Personalization |
| **Effort** | Customization, creation | Ownership |
| **Social** | Connections, followers | Network effects |

**Investment improves future triggers** — the cycle continues.

### Hook Model Audit

```
TRIGGER
□ What external triggers drive users to your product?
□ What internal triggers could associate with your product?
□ Are you building toward internal triggers?

ACTION
□ What is the simplest action users take?
□ Have you removed all unnecessary friction?
□ Is the action triggered at the right moment?

REWARD
□ What variable reward do users receive?
□ Is there variety (tribe/hunt/self)?
□ Is the reward fulfilling the user's need?

INVESTMENT
□ What do users put into the product?
□ Does investment load the next trigger?
□ Does investment increase stored value?
```

### Ethical Considerations

The Hook Model can be used for good (helping people form healthy habits) or harm (creating addictive products).

**Manipulation Matrix**:

| | Maker uses product | Maker doesn't use product |
|---|---|---|
| **Materially improves user's life** | Facilitator (ethical) | Peddler (questionable) |
| **Doesn't improve user's life** | Entertainer (neutral) | Dealer (unethical) |

**Questions to ask**:
- Would I use this product myself?
- Does this genuinely improve users' lives?
- Am I creating value or exploiting vulnerabilities?

## Combining Frameworks

### LIFT + Fogg for Landing Pages

1. **Audit with LIFT** to identify conversion barriers
2. **Apply Fogg** to reduce friction in key actions
3. **Prioritize** based on impact vs. effort

### Fogg + Hook for Product Design

1. **Design actions** using Fogg (B=MAP)
2. **Build habit loops** using Hook Model
3. **Measure** trigger → action completion rates

### Full Optimization Flow

```
DIAGNOSE (LIFT)
    │
    ├─► What's the barrier?
    │   - Value unclear? → Improve value prop
    │   - Relevance issue? → Better targeting
    │   - Clarity problem? → Simplify design
    │   - Anxiety? → Add trust signals
    │   - Distraction? → Remove elements
    │
    ▼
FIX FRICTION (FOGG)
    │
    ├─► Increase motivation or ability?
    │   - Usually increase ability (easier)
    │   - Motivation is expensive to change
    │
    ▼
BUILD HABIT (HOOK)
    │
    └─► If retention is the goal:
        - Map the trigger → action → reward → investment
        - Reduce time between cycles
        - Add variability to rewards
```

## Implementation Checklist

### Quick LIFT Audit
- [ ] Can someone understand the offer in 5 seconds?
- [ ] Is there one clear CTA?
- [ ] Are trust signals visible?
- [ ] Is there a reason to act now?

### Quick Fogg Check
- [ ] Is the target action as simple as possible?
- [ ] Are prompts delivered at the right time?
- [ ] Is motivation supported by the design?

### Quick Hook Assessment
- [ ] What triggers bring users back?
- [ ] What's the simplest action they take?
- [ ] What variable reward do they receive?
- [ ] What do they invest that increases loyalty?
